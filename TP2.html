2. Rappel sur le patron Singleton

Design pattern : Singleton

Singleton garantit qu’une classe n’a qu’une seule instance et fournit un point d’accès global à cette instance.

4. Le patron Fabrique (simple)

Nous y sommes, vous venez de décortiquer le patron Fabrique Simple

Design pattern : Fabrique (simple)

Fabrique (simple) définit une interface pour la création d’un objet, mais en laissant à des sous-classes le choix des classes à instancier (voir aussi Fabrique abstraite).

Figure 4. Modèle UML du patron Fabrique

QUESTION

  1. Implémentez complètement l’application de Pizzeria de manière à ce que le programme de test ci-dessous produise le résultat ci-dessous. section 4.2 du TD2.

    Commencez par écrire ce programme et servez-vous des QuickFix pour "générer" un maximum de code.

  2. Générez votre diagramme de classe automatiquement

On utilisera le modèle suivant pour les pizzas :

Programme de test
public class PizzaTestDrive {
  public static void main(String[] args) {
    Pizzeria boutiqueBrest = new PizzeriaBrest();
    Pizzeria boutiqueStrasbourg = new PizzeriaStrasbourg();

    Pizza pizza = boutiqueBrest.commanderPizza("fromage");
    System.out.println("JMB a commandé une " + pizza.getNom() + "\n");

    pizza = boutiqueStrasbourg.commanderPizza("fromage");
    System.out.println("JMI a commandé une " + pizza.getNom() + "\n");
  }
}
